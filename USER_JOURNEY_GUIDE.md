# 🗺️ User Journey Guide - 4 New Features

## Overview
This guide shows exactly how users will discover and use each new feature.

---

## 📄 Feature 1: PDF Health Report

### User Journey Map
```
Login → Dashboard → "Premium & Postpartum" Section → "PDF Report 👑" 
  → Page Loads → Click "Generate PDF Report" 
    → [Premium Check]
      → ✅ Premium User: Loading... → PDF Downloads → Open & Share
      → ❌ Free User: Upgrade Prompt → Premium Page
```

### Discovery Paths
1. **Primary**: Dashboard → "Premium & Postpartum" grid
2. **Secondary**: More Menu → "PDF Report"
3. **Tertiary**: Push notification → Direct link

### User Scenarios

#### Scenario A: Before Doctor Visit
```
User: "I have a doctor appointment tomorrow"
  → Opens app
  → Sees "PDF Report" tile
  → Clicks it
  → Generates PDF
  → Shares via WhatsApp with doctor
  → Doctor appreciates organized data
  → Better consultation experience
```

#### Scenario B: Premium Gate Conversion
```
Free User: Clicks "Generate PDF"
  → Alert: "This is a Premium feature"
  → Shows upgrade button
  → User sees value (needs it for tomorrow's appointment)
  → Upgrades to Premium
  → Returns to PDF page
  → Generates report successfully
  → Conversion! 🎉
```

### Key Touchpoints
- **Discovery**: Dashboard tile with 👑 icon (signals premium)
- **Action**: One-click generation (low friction)
- **Value**: Immediate download (instant gratification)
- **Retention**: User returns before each doctor visit

## 🩺 Feature 2: ASHA/ANM Chatbot Mode

### User Journey Map
```
Health Worker Login → Dashboard → Sees "ASHA Mode" 
  → Clicks → Info Screen (explains purpose)
  → Selects Language (Hinglish/Hindi/English)
  → Sees 8 Quick Topics
  → Clicks "Anemia Detection" OR Types Custom Query
  → AI Response Appears
    → Urgency Level
    → Assessment
    → Action Steps
    → Red Flags
    → Counseling Tips
  → Takes Action (refers patient OR provides guidance)
```

### Discovery Paths
1. **Primary**: Dashboard → "Premium & Postpartum" → "ASHA Mode"
2. **Secondary**: More Menu → "ASHA Assistant"
3. **Tertiary**: Direct link from government training materials

### User Scenarios

#### Scenario A: Field Health Worker
```
ASHA Worker: Visits pregnant woman in village
  → Woman complains of breathlessness
  → Worker opens ASHA Mode
  → Selects Hindi language
  → Clicks "High-Risk Pregnancy Signs"
  → AI responds: "URGENT - refer to PHC immediately"
  → Shows warning signs to watch for
  → Worker refers patient
  → Potential complication prevented ✅
```

#### Scenario B: ANM Training Session
```
ANM Supervisor: Training 20 ASHA workers
  → Demonstrates ASHA Mode feature
  → Shows how to ask questions in Hindi
  → Workers practice with sample scenarios
  → Supervisor shares app link
  → Workers download app
  → 20 new B2G users acquired 🎯
```

### Key Touchpoints
- **Discovery**: Blue badge on dashboard (different from pink theme)
- **Trust**: Info card explains it's for health workers
- **Action**: Quick topics = low barrier to first query
- **Value**: Instant AI guidance in local language
- **Retention**: Worker uses it 5-10x per week in field

## 🍼 Feature 3: Breastfeeding Tracker

### User Journey Map
```
New Mom Login → Dashboard → "Premium & Postpartum" → "Breastfeeding"
  → Page Loads → Sees 3 Buttons (Left/Right/Both)
  → Clicks "Start Feed - Left"
  → Timer Starts (00:01... 00:02...)
  → Baby Switches → Clicks "Switch to Right"
  → Timer Continues
  → Feed Ends → Clicks "End Feed"
  → Modal: Rate Latch Quality + Add Notes
  → Saves Feed
  → Summary Updates: "3 feeds today, 45 min total"
  → Sees Feed History
  → Checks Weekly Chart
```

### Discovery Paths
1. **Primary**: Dashboard → "Premium & Postpartum" → "Breastfeeding"
2. **Secondary**: More Menu → "Breastfeeding"
3. **Tertiary**: Push notification after delivery date
4. **Organic**: New mom asks "How to track breastfeeding?"

### User Scenarios

#### Scenario A: First-Time Mom
```
New Mom (Day 3 postpartum): Confused about feeding frequency
  → Doctor said "feed every 2-3 hours"
  → Opens breastfeeding tracker
  → Starts logging each feed
  → After 1 day, sees pattern:
    - 8 feeds in 24 hours ✅
    - Avg duration 15 min
    - Last feed 2h ago → Time for next feed!
  → Feels more confident
  → Shows doctor at next visit
  → Doctor praises organized tracking
```

#### Scenario B: Lactation Consultant Visit
```
Mom: "Baby isn't gaining weight"
  → LC asks: "How often are you feeding?"
  → Mom: "Let me check my tracker..."
  → Opens feed history
  → Shows: Only 4-5 feeds/day (should be 8-12)
  → LC: "That's the issue!"
  → Adjusts feeding schedule
  → Problem solved with data! 📊
```

### Key Touchpoints
- **Discovery**: Shows up after user's due date passes
- **First Use**: Simple 3-button UI (no learning curve)
- **Habit Formation**: Used 8-12x per day for 6+ months
- **Value**: Peace of mind + concrete data for doctors
- **Retention**: Highest daily engagement of any feature

## 📧 Feature 4: Weekly Email Digest

### User Journey Map
```
User: Goes about her week
  → Friday: Logs weight, mood, sleep
  → Saturday: Schedules appointment
  → Sunday 8 AM: Email arrives!
  → Subject: "Your pregnancy week 24 summary 🌸"
  → Opens Email
    → Beautiful gradient design
    → "You're 24 weeks pregnant"
    → Weight trend chart
    → Mood summary: "Mostly happy this week 😊"
    → Sleep quality: 6.5 hrs avg
    → Upcoming appointment reminder
    → CTA: "Open App →"
  → Clicks link
  → App opens to dashboard
  → Engagement loop continues
```

### Discovery Paths
1. **Primary**: Automatic (enabled by default)
2. **Secondary**: Settings page → Email preferences
3. **Awareness**: Welcome email mentions weekly digests

### User Scenarios

#### Scenario A: Inactive User Re-engagement
```
User: Hasn't opened app in 10 days
  → Sunday: Email arrives
  → Subject line catches attention
  → Opens email
  → Sees: "You haven't logged weight in 10 days"
  → Feels gentle nudge (not guilt)
  → Clicks "Log Weight Now"
  → Returns to app
  → Logs weight, checks kick counter
  → Re-engaged! 🔄
```

#### Scenario B: Family Sharing
```
Pregnant Woman: Receives weekly email
  → Forwards to husband:
    "Look honey, here's my week summary!"
  → Husband sees:
    - Week 32 progress
    - Baby weight estimate
    - Upcoming doctor visit
  → Husband feels more connected
  → Asks: "How are you feeling?"
  → Couple bonding moment 💕
```

#### Scenario C: Opt-out Flow
```
User: "Too many emails"
  → Opens email
  → Scrolls to bottom
  → Clicks "Unsubscribe" or "Manage preferences"
  → Opens settings page
  → Sees options:
    - Disable digest
    - Change day (Sunday → Saturday)
    - Change time (8 AM → 7 PM)
  → User keeps it but changes to Saturday evening
  → Opt-out prevented! ⚙️
```

### Key Touchpoints
- **Timing**: Sunday morning (low inbox competition)
- **Design**: Beautiful, not spammy
- **Content**: Personalized with real data
- **CTA**: Clear "Open App" button
- **Frequency**: Weekly (not overwhelming)
- **Value**: Reflection + motivation

## 🎨 UI/UX Patterns

### Common Patterns Across All Features
1. **Discovery**: Consistent placement (dashboard + more menu)
2. **Entry**: Emoji + clear label (📄 PDF Report, 🍼 Breastfeeding)
3. **First Screen**: Explanation card (what it does, why it's useful)
4. **Primary Action**: Big, colorful button (hard to miss)
5. **Feedback**: Loading states, success messages, error handling
6. **Empty State**: Graceful messaging when no data exists

### Mobile-First Design
- All features work on small screens (375px width)
- Touch targets are 44px minimum
- Forms use native inputs (date, time, select)
- Charts are responsive (Chart.js)
- Buttons are thumb-friendly (bottom of screen)
